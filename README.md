# ⬡ PyVault

**Локальное хранилище Python-скриптов с браузерным интерфейсом.**

Запускаете один сервер — получаете удобную среду для хранения, организации и запуска своих скриптов прямо в браузере. Никаких облаков, никаких аккаунтов — всё на вашем компьютере.

---

## Быстрый старт

**Требования:** Python 3.8+, Flask

```bash
pip install flask
python app.py
```

Браузер откроется автоматически на `http://localhost:7332`

## Возможности

### Скрипты

- **Создание** — кнопка `＋ Скрипт` в тулбаре или через контекстное меню холста
- **Редактор** — встроенный редактор с подсветкой синтаксиса Python, нумерацией строк и автоотступами
- **Запуск** — кнопка `▶ Запустить` внутри редактора или с карточки; перед запуском запрашивается рабочая директория
- **Консоль** — вывод скрипта в реальном времени с поддержкой цветов ANSI и кириллицы
- **Остановка** — кнопка `⏹ Остановить` прерывает выполнение
- **Закрепление** — скрипты можно закрепить (`📌`), они отображаются первыми
- **Дублирование** — через контекстное меню
- **Компиляция в EXE** — через PyInstaller (требует установки); поддержка `onefile` и `onedir`, опция `--noconsole`, своя иконка `.ico`

### Папки

- **Вложенность** — папки могут содержать подпапки любой глубины
- **Дерево в сайдбаре** — раскрываемое дерево со счётчиками скриптов
- **Редактирование** — переименование, смена иконки и цвета через контекстное меню
- **Перемещение** — перетаскивание папок и скриптов мышью (drag & drop), в том числе на элементы сайдбара
- **Корзина** — удалённые папки и скрипты попадают в корзину и могут быть восстановлены

### Проекты

- **Формат** — проект сохраняется в файл `.pyvault` (JSON)
- **Открытие** — поддержка File System Access API (Chrome/Edge): файл привязывается, автосохранение работает напрямую
- **Недавние** — список последних открытых проектов с быстрым повторным открытием
- **Новый проект** — при каждом запуске приложения открывается пустой проект и показывается список недавних
- **Индикатор изменений** — точка `●` в заголовке сигнализирует о несохранённых правках

### Зависимости

- Вкладка `📦 Зависимости` в редакторе анализирует `import`-ы скрипта
- Показывает сторонние пакеты, стандартную библиотеку, функции и классы
- Установка недостающих пакетов одной кнопкой прямо из интерфейса (`pip install`)

### Интерфейс

| Возможность | Описание |
|---|---|
| 4 режима отображения | Сетка, Крупные карточки, Плитки, Компактный, Список |
| Сортировка | По дате изменения, имени или количеству строк |
| Поиск | Мгновенный поиск по имени, описанию и тегам |
| Боковая панель | Изменяемая ширина (перетаскивание разделителя) |
| Хлебные крошки | Навигация по вложенным папкам |
| Тёмная тема | Единственная и постоянная — оптимизирована для долгой работы |
| Запоминание вида | Выбранный режим отображения сохраняется между сессиями |

## REST API

Сервер предоставляет внутренний API, на котором построен весь интерфейс.

### Состояние

| Метод | URL | Описание |
|---|---|---|
| `GET` | `/api/state` | Получить текущее состояние проекта |
| `POST` | `/api/state` | Обновить поля состояния |
| `POST` | `/api/project/new` | Создать новый пустой проект |
| `GET` | `/api/project/export` | Скачать проект как `.pyvault` |
| `POST` | `/api/project/import` | Загрузить `.pyvault` файл |

### Скрипты

| Метод | URL | Описание |
|---|---|---|
| `POST` | `/api/script` | Создать скрипт |
| `PUT` | `/api/script/<id>` | Обновить поля скрипта |
| `DELETE` | `/api/script/<id>` | Удалить скрипт |
| `POST` | `/api/script/<id>/run` | Запустить скрипт |
| `POST` | `/api/script/<id>/stop` | Остановить скрипт |
| `GET` | `/api/script/<id>/logs` | Получить вывод и статус выполнения |
| `GET` | `/api/script/<id>/analyze` | Анализ зависимостей и структуры кода |

### Папки

| Метод | URL | Описание |
|---|---|---|
| `POST` | `/api/folder` | Создать папку (поддерживает `parent_id`) |
| `PUT` | `/api/folder/<id>` | Обновить папку (имя, иконка, цвет, `parent_id`) |
| `DELETE` | `/api/folder/<id>` | Удалить папку |

### Утилиты

| Метод | URL | Описание |
|---|---|---|
| `POST` | `/api/install` | Установить pip-пакет |
| `POST` | `/api/compile/<id>` | Скомпилировать скрипт в EXE через PyInstaller |
| `GET` | `/api/ls?path=...` | Листинг директории (для выбора рабочей папки) |
| `GET` | `/api/drives` | Список дисков (Windows) |

## Заметки

- Все данные хранятся **в памяти** до явного сохранения в `.pyvault`. После перезапуска `app.py` несохранённый проект теряется.
- Корзина хранится в `localStorage` браузера — очищается если очистить данные сайта.
- Сервер запускается на `0.0.0.0:7332` — доступен в локальной сети. Не рекомендуется открывать наружу без аутентификации.
- На Windows для корректного отображения Unicode в консоли используется автоматическая перекодировка stdout/stderr в UTF-8.
